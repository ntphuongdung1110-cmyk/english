## 🗣️ Mẫu Câu — Writing & Saying Performance Reviews

### Summarizing Performance
```
"Overall, [Name] has had a strong quarter. 
Key strengths include [strength 1] and [strength 2].
The main area for development is [area], 
which I'd like us to focus on in Q[X]."
```

### Setting Goals for Next Period
```
"For the next quarter, I'd like to see [specific goal].
I'll support this by [how you'll help].
We'll check in monthly to track progress."
```

### Giving Rating With Context
```
"I'm rating this as 'Meets Expectations' — 
and I want to be clear that this is a strong rating.
The one area keeping us from 'Exceeds' is [specific thing].
Here's what it would take to get there: [clear path]."
```

## 🎧 Shadowing Script — Performance Review Conversation

```
"Thanks for making time for this review.
I want to make this a two-way conversation — 
so please feel free to push back on anything.

Overall, I think this has been a solid quarter for you.
Your test coverage has improved significantly, 
and the quality of your bug reports is noticeably better.
That's real, measurable progress.

The area I want us to focus on going forward 
is communication — specifically, 
how you communicate risk to non-technical stakeholders.
I've noticed that when you present to the PM, 
the technical details sometimes overshadow the key message.

So for Q2, I'd like you to practice structuring updates 
with the bottom line first — lead with the impact, 
then the details.

I'll help by doing a practice session before 
each major presentation. Is that something you're open to?

And — I want to hear from you. 
How do you feel about the quarter? 
What do you think went well? What do you wish you'd done differently?"
```
