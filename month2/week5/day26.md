## 📚 Từ Vựng Hôm Nay — Facilitation Language

### Mời Tham Gia
```
"I'd love to hear [name]'s perspective on this."
"Does anyone have a different take?"
"We haven't heard from [team/person] yet — any thoughts?"
"I want to make sure all voices are heard."
"[Name], what's your read on this?"
```

### Kéo Discussion Về Track
```
"That's a great point — let's park that and come back."
"I want to make sure we stay on the main question."
"Let's try to bring this back to [topic]."
"We're running short on time — let's focus on [decision]."
"Can we take that offline? I want to get to [next item]."
```

### Tổng Hợp Ý Kiến
```
"So it sounds like there are two camps here: [view A] and [view B]."
"Let me see if I can summarize the key points so far."
"We seem to be aligned on [X], but there's still debate on [Y]."
"I'm hearing [theme] as a common concern — is that right?"
```

### Dẫn Đến Quyết Định
```
"We need to make a call on this — let's do a quick show of hands."
"Can we agree to [compromise] for now?"
"Who has the final say on this one?"
"Let's go with [option] — we can revisit if needed."
```

## 🗣️ Mẫu Câu + Ví Dụ Thực Tế

### Pattern 1: Khi 1 người độc chiếm thảo luận
```
"Great input — [Name], let me pause you there for a second 
and see if others have thoughts. [Other Name], what do you think?"
```

### Pattern 2: Khi discussion quá im lặng
```
"Let me try a different approach — 
what would each team ideally want to see happen here?
Dev team first, then QC, then PM."
```

### Pattern 3: Khi có 2 quan điểm đối lập
```
"So we have two perspectives on the table.
On one side: [view A]. On the other: [view B].
Rather than debate which is right, 
can we focus on: what outcome do we both want?
Because I think the goal is the same — [shared goal]."
```

## 🎧 Shadowing Script — Facilitation in Action

```
"Okay, so we've heard a few different views on this.
Let me try to summarize where we're at.

It sounds like the dev team is concerned about timeline, 
and QC is concerned about quality risk. 
Both are valid. 

So the question isn't 'who's right' — 
it's 'what's the best decision given the constraints?'

[Name from dev], what would you need 
to feel comfortable with a 24-hour delay?
And [QC person], if we did release on time, 
what's the minimum testing you'd need to feel okay?

Let's see if there's a middle ground there."
```
