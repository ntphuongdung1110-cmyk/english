## 📚 Từ Vựng Hôm Nay — QC Tech & Automation

### Tools & Frameworks
| Từ / Cụm | Nghĩa / Dùng Khi |
|---|---|
| **test automation framework** | khung tự động hóa kiểm thử |
| **CI/CD pipeline** | quy trình tích hợp & triển khai liên tục |
| **regression suite** | bộ test hồi quy tự động |
| **test coverage** | % code hoặc feature được test |
| **flaky test** | test không ổn định, lúc pass lúc fail |
| **smoke test** | test nhanh xem build có chạy được không |
| **sanity test** | test nhỏ sau khi fix bug |
| **load testing** | test hiệu suất khi nhiều user cùng lúc |
| **API testing** | kiểm thử giao tiếp giữa services |
| **end-to-end (E2E) testing** | test toàn bộ luồng từ đầu đến cuối |

### Talking About Automation
```
"We've been automating our regression suite using [tool]."
"Right now our automation coverage is around [X]%."
"We're looking to integrate testing into the CI/CD pipeline."
"One challenge we have is flaky tests — they pass locally but fail in CI."
"We're evaluating [tool A] vs [tool B] for our next automation framework."
"The return on investment for automation really shows after [timeframe]."
```

## 🗣️ Mẫu Câu Hôm Nay

### Pattern 1: Mô tả tech stack hiện tại
```
"Currently, our QC setup uses [tool] for [purpose] 
and [tool] for [purpose]. 
It's been working well for [use case], 
though we're running into some challenges with [pain point]."
→ Currently, our QC setup uses Selenium for regression testing 
   and Postman for API testing. It's been working well for web flows, 
   though we're running into some challenges with test maintenance.
```

### Pattern 2: Đề xuất automation investment
```
"Right now, [X]% of our tests are manual. 
If we invest in automating [specific area], 
I estimate we'd save [time/effort] per sprint 
and catch [type of issues] earlier."
```

### Pattern 3: Giải thích kỹ thuật cho non-tech audience
```
"Think of [technical concept] like [simple analogy].
Basically, [simple explanation]."
→ Think of CI/CD like an assembly line for software. 
   Basically, every time a developer pushes code, 
   our automated tests run automatically to catch issues right away.
```

### Pattern 4: Thảo luận về automation challenges
```
"One thing we're struggling with is [challenge].
The impact is [effect on team/process].
I've been looking into [potential solution] as a way to address it."
→ One thing we're struggling with is flaky tests — 
   they pass on my machine but fail in the CI pipeline. 
   The impact is we lose trust in the test results. 
   I've been looking into test retry mechanisms as a way to address it.
```

## 🎧 Shadowing Script — Tech Discussion

```
"So, when it comes to our automation strategy, 
here's where we're at.

We started building our automation framework 
about 6 months ago, using Cypress for E2E testing. 
Currently we're at around 35% automation coverage — 
which honestly isn't where I'd like it to be, 
but it's realistic given the team size.

The main challenge right now is what we call 'flaky tests' — 
tests that are unreliable. They pass sometimes, fail other times, 
for no obvious reason. 
It erodes confidence in the test suite.

What I'm planning to do is — 
first, identify and quarantine the flaky ones, 
then gradually fix the root causes, 
and in parallel, continue expanding coverage 
for the high-risk areas like payment and authentication.

By end of Q2, I'm aiming for 55% coverage. 
It's ambitious, but I think it's achievable."
```
