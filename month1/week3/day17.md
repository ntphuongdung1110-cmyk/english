## 📚 Từ Vựng Hôm Nay

| Từ / Cụm | Nghĩa | Ví Dụ |
|---|---|---|
| **have a candid conversation** | nói chuyện thẳng thắn | I'd like to have a candid conversation about this. |
| **address the elephant in the room** | nói về vấn đề ai cũng biết nhưng né | Let's address the elephant in the room. |
| **I take responsibility** | tôi chịu trách nhiệm | I take responsibility for missing that deadline. |
| **I'm not trying to point fingers** | không muốn đổ lỗi ai | I'm not trying to point fingers — I just want to understand. |
| **this isn't working for me** | tình trạng này không ổn với tôi | Honestly, this current process isn't working for me. |
| **I need to be direct** | cần nói thẳng | I need to be direct about something. |
| **moving forward** | từ đây về sau | How can we make sure this doesn't happen moving forward? |
| **acknowledge** | thừa nhận | I want to acknowledge that this has been frustrating. |
| **find a middle ground** | tìm điểm chung | I think we can find a middle ground here. |
| **unacceptable** | không thể chấp nhận | Releasing with known critical bugs is unacceptable. |

## 🗣️ Mẫu Câu — 3 Tình Huống Khó Thường Gặp

### Tình Huống 1: Dev liên tục push back bug reports của bạn
```
"I want to have a candid conversation about something 
that's been affecting our workflow.

I've noticed that several of my bug reports have been 
marked as 'not a bug' or closed without investigation. 
I'm not trying to point fingers — I genuinely want to 
understand if there's a gap in how I'm documenting issues.

But I also want to be transparent: when valid bugs get dismissed, 
it puts us at risk. Can we find a better process together?"
```

### Tình Huống 2: PM muốn release dù QC chưa sign off
```
"I need to be direct about something before we finalize the release.

We have 2 open critical bugs that haven't been fixed yet.
Releasing in this state is a risk I'm not comfortable signing off on.

I understand there's timeline pressure — I really do.
But if these hit production, the impact on users will be significant.

Can we get 12 hours to resolve these? 
I'm happy to have QC working through the night if needed.
I just need us to agree this is the right call."
```

### Tình Huống 3: Team member không làm đúng quy trình
```
"Hey, do you have a minute? I want to address something directly.

I noticed that the last 3 test cycles were run without 
updating the test documentation afterward.

I know it feels like extra work, but that documentation 
is what the next team member relies on. 
When it's missing, it slows everyone down.

I'm not looking to make this a big thing — 
I just need this to be consistent going forward. 
Can I count on that?"
```

## 🎧 Shadowing Script

```
"Look, I want to be straightforward with you 
because I think we both want the same thing here — 
a clean, stable release.

Here's where I'm at: I have real concerns 
about the current state of the build. 
We found two critical bugs yesterday that are 
not yet resolved, and releasing with those in place 
is something I can't support.

I want to acknowledge that this puts you in a tough spot, 
and I'm not saying this to create friction. 
I'm saying it because it's my job to protect the product — 
and ultimately, to protect the users.

So here's what I'd like to propose. 
Can we give the dev team until tomorrow noon to fix these? 
QC will run a targeted regression immediately after. 
We're talking maybe a 24-hour delay — 
and I think it's worth it.

I'm open to hearing your perspective, 
but I did want to be clear about where I stand."
```

## 💬 Speaking Prompt — Ghi Âm 3 Phút

> Chọn 1 tình huống thực tế bạn đã gặp (hoặc dùng tình huống giả):
> **"A senior developer keeps closing your bug reports as 'by design' without proper investigation. You need to address this."**

**Yêu cầu:**
- Tone: direct nhưng respectful
- Không buộc tội — đặt câu hỏi trước
- Đề xuất process cụ thể để tránh lặp lại
- Kết bằng: mở hướng hợp tác
