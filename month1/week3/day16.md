## 📚 Từ Vựng Hôm Nay — Presentation & Data Language

### Signposting (Dẫn Dắt Người Nghe)
| Cụm | Dùng Khi | Ví Dụ |
|---|---|---|
| **I'd like to walk you through** | Mở đầu | I'd like to walk you through last month's QC results. |
| **Let me start with** | Bắt đầu phần đầu | Let me start with the overall test summary. |
| **Moving on to** | Chuyển phần | Moving on to defect analysis. |
| **As you can see** | Chỉ vào data/chart | As you can see, our pass rate improved by 12%. |
| **This is significant because** | Giải thích tầm quan trọng | This is significant because it reflects our shift-left approach. |
| **To summarize** | Tóm tắt | To summarize, we closed 47 of 52 bugs this sprint. |
| **Based on these results** | Rút ra kết luận | Based on these results, I'd recommend extending UAT. |
| **I'd like to leave you with** | Kết | I'd like to leave you with one key takeaway. |

### Mô Tả Số Liệu
```
"increased by [X]% / decreased by [X]%"
"up from [X] to [Y]"
"down from [X] to [Y]"
"roughly / approximately [X]"
"account for [X]% of total"
"compared to last [week/month/sprint]"
"the breakdown is as follows"
"the majority of / most of"
```

## 🗣️ Mẫu Câu Hôm Nay

### Pattern 1: Mở Bài
```
"I'd like to walk you through our QC results for [period].
I'll cover three areas: [area 1], [area 2], and [area 3].
This should take about [X] minutes."
```

### Pattern 2: Trình Bày Số Liệu
```
"As you can see, we ran [X] test cases this sprint. 
[Y] passed, [Z] failed, and [N] are still in progress.
That gives us an overall pass rate of [X]%, 
which is up from [Y]% last sprint."
```

### Pattern 3: Phân Tích + Ý Nghĩa
```
"The most notable finding is [observation].
This is significant because [reason/impact].
The root cause appears to be [cause]."
```

### Pattern 4: Recommendation
```
"Based on these results, I'd recommend [action].
If we implement this, I expect [outcome] by [timeframe]."
```

### Pattern 5: Kết Bài + Q&A
```
"To summarize: [3 key points].
I'd like to leave you with one key takeaway: [main message].
I'm happy to take any questions now."
```

## 🎧 Shadowing Script — QC Monthly Report

```
"Good morning everyone. I'd like to walk you through 
our QC results for March.

I'll cover three things: overall test summary, 
defect analysis, and my recommendations going forward.

Let me start with the test summary. 
This sprint, we ran 156 test cases across 4 modules. 
128 passed, 19 failed, and 9 are still in progress. 
That gives us a pass rate of 82% — 
which is up from 74% last sprint. Good progress.

Moving on to defect analysis. 
We logged 23 new bugs. Of those, 
17 were resolved before go-live — that's 74%. 
The remaining 6 are low-severity and have been 
documented for the next sprint.

As you can see, the majority of bugs — 
about 65% — were found in the payment module. 
This is significant because it's our highest-risk area, 
and it confirms we need more test coverage there.

Based on these results, I'd recommend 
allocating 2 more automation test cases to payment flows next sprint.

To summarize: pass rate improved, defect closure rate is strong, 
and we have a clear focus area for next sprint.

Happy to take any questions."
```
