## 📊 Bài Kiểm Tra Cuối Tuần 1

### Test 1: Từ Vựng (không nhìn bảng)

Điền vào chỗ trống:

```
1. I need to ________ with the dev lead about this bug. (kéo vào)
   → loop in

2. As far as I'm ________, this is a critical issue. (quan tâm/liên quan)
   → concerned

3. The ________ cause of this bug was an API timeout. (nguyên nhân)
   → root

4. We need ________ from the PM before we can release. (phê duyệt)
   → sign-off

5. I can ________ this bug on iOS 17 every time. (tái hiện)
   → reproduce

6. We should ________ left — involve QC from the planning stage. (chuyển)
   → shift

7. There are 3 open ________ in the checkout flow. (lỗi/defect)
   → defects / bugs

8. At the end of the ________, quality is what users experience. (idiom kết luận)
   → day
```

### Test 2: Nghe (không phụ đề)

Bật 1 video Brian bất kỳ trong 3 phút:
```
□ Hiểu được: ___% nội dung
□ Ghi lại 5 từ/cụm bạn nghe được
□ Ghi lại 2 filler words Brian dùng
```

### Test 3: Nói (ghi âm 5 phút)

> **"Imagine you're being interviewed for a senior QC Manager position. Tell me about yourself, describe your typical day, share your opinion on the biggest challenge in QC, and give me an example of a time you handled a critical bug."**

**Cấu trúc 5 phút:**
```
Phút 1:   Self-introduction (dùng từ Day 1)
Phút 2:   Daily routine (dùng action verbs Day 2)
Phút 3:   Opinion on QC challenges (dùng opinion starters Day 3)
Phút 4:   Bug story + filler words (Day 4 + Day 5)
Phút 5:   Kết thúc tự nhiên
```
