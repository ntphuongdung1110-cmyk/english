## 📚 Từ Vựng QC Chuyên Ngành

### Nhóm 1: Bug & Defect
| Từ / Cụm | Nghĩa | Ví Dụ |
|---|---|---|
| **defect / bug** | lỗi phần mềm | We found a critical defect in the payment flow. |
| **severity** | mức độ nghiêm trọng | This bug has Severity 1 — it blocks core functionality. |
| **priority** | mức độ ưu tiên | High priority — needs to be fixed before tomorrow's release. |
| **root cause** | nguyên nhân gốc rễ | The root cause was an unhandled null value. |
| **reproduce** | tái hiện lỗi | I can reproduce this bug consistently on iOS 17. |
| **regression** | lỗi cũ xuất hiện lại | This looks like a regression from the last deployment. |
| **workaround** | cách xử lý tạm | There's a workaround, but we need a proper fix. |
| **edge case** | trường hợp đặc biệt | This only happens in edge cases, but we still need to address it. |

### Nhóm 2: Process & Planning
| Từ / Cụm | Nghĩa | Ví Dụ |
|---|---|---|
| **test coverage** | độ bao phủ kiểm thử | Our test coverage is at 75% — we need to improve. |
| **test case** | kịch bản kiểm thử | I've written 40 test cases for this sprint. |
| **sign-off** | phê duyệt | We need sign-off from the PM before release. |
| **UAT** | User Acceptance Testing | UAT starts next Monday with the client team. |
| **release candidate** | phiên bản dự kiến phát hành | The release candidate is ready for final QC review. |
| **go-live** | ngày phát hành | Go-live is scheduled for Friday. |
| **bottleneck** | điểm nghẽn | The bottleneck right now is the automation setup. |
| **shift left** | kiểm thử sớm trong quy trình | We're shifting left — QC is involved from sprint planning now. |

## 🗣️ Mẫu Câu Trong Meeting QC

### Pattern 1: Báo cáo bug
```
"I'd like to flag a [severity] issue we found in [feature/module]."
→ I'd like to flag a critical issue we found in the checkout flow.
→ I need to raise a concern about a bug we're seeing in the login screen.
```

### Pattern 2: Mô tả bug
```
"What happens is [description]. It occurs when [condition]."
→ What happens is the app crashes. It occurs when the user tries to pay with an expired card.
→ What we're seeing is incorrect data being displayed. It happens when the user switches accounts.
```

### Pattern 3: Đề xuất hành động
```
"My recommendation would be to [action] before [deadline/milestone]."
→ My recommendation would be to delay the release by one day to fix this properly.
→ I'd suggest we get the dev team to look at this before we push to staging.
```

### Pattern 4: Hỏi về fix/timeline
```
"Do we have an ETA on the fix for [issue]?"
"When can we expect this to be resolved?"
→ Do we have an ETA on the fix for the login bug?
→ Is there any update on when the payment issue will be addressed?
```

### Pattern 5: Cập nhật trạng thái QC
```
"As of [time/date], [X] of [Y] test cases have passed. 
We're currently blocked by [issue]."
→ As of this morning, 38 of 42 test cases have passed. 
   We're currently blocked by the API timeout issue.
```

## 🎧 Shadowing Script — QC Status Update

```
"Okay, so here's where we stand on QC for this release.

As of this morning, we've completed testing on the 
core features — login, dashboard, and user settings. 
All test cases passed there.

However, we did flag two issues in the checkout flow. 
One is a Severity 2 bug — it doesn't block checkout 
completely, but it causes incorrect pricing to show 
under certain conditions. We've already logged this 
and the dev team is looking into it.

The second issue is a visual bug — low severity, 
but it's noticeable on Android. We'd recommend 
fixing that before go-live if time allows.

Overall, we're on track, but I'd like to do one 
more regression run on Friday before we sign off. 
Does that work for everyone?"
```

## 💬 Speaking Prompt — Roleplay (Ghi Âm 3 Phút)

> **Tình huống:** Bạn đang trong sprint review meeting. PM hỏi: "Can you give us a quick QC update?"

**Yêu cầu:**
- Báo cáo trạng thái test (bao nhiêu test cases pass/fail)
- Nêu ít nhất 1 bug critical với severity + mô tả ngắn
- Đề xuất hành động cụ thể
- Dùng ít nhất 5 từ QC chuyên ngành từ bảng trên

## 🔄 Pronunciation Lưu Ý

```
"severity"   → /sɪˈvɛrɪti/ (không phải "se-VE-ri-ty")
"prioritize" → /praɪˈɒrɪtaɪz/
"regression" → /rɪˈɡrɛʃən/
"scenario"   → /sɪˈnɑːrioʊ/ (Mỹ) hoặc /sɪˈnɛərioʊ/ (Anh)
"UAT"        → đọc từng chữ: U-A-T
"ETA"        → đọc từng chữ: E-T-A
```
